Hamachi assigns 0.0.0.0 IP addresses to all peers

Explanation:

This can sometimes occur when Windows does not properly "bridge" the Hamachi adapter with the Internet adapter. This can happen at installation, during Windows updates, or randomly.

Solution: Rebridge the adapters

  1. Open the run prompt Win + r
  2. Enter ncpa.cpl to open Network Connections
  3. You should see an adapter named Hamachi as well as Hamachi - ###-###-###
    4. Pro tip: You can tell the adapter is not bridged correctly if their Internet adapter does not say Enabled, Bridged
  4. The Hamachi - ###-###-### adapter is bridged with their regular internet adapter (typicaly labeled Ethernet or WiFi)
  5. To re-bridge these adapters, hold Ctrl and click both Hamachi - ###-###-### and Ethernet (or similar)
  6. Now, right click one of the adapters and select "Remove from bridge"
  7. Repeat step 5, this time right-clicking and selecting Bridge Connections
  8. The internet will likely disconnect temporarily at this point, wait for it to come back
  9. Re-test
Note! If you receive an error at any point, or this does not resolve the 0.0.0.0 IP issue, reach out to a SME.

%40 Catalina Issue (Client)

Explanation:

This occurs due to an error in a component of the program on Mac computers and will be fixed by a future update of the Mac software.

Solution: Clear out the program cache and re-try

  1. Important! Close the LogMeIn Client completely before continuing
  2. Open the Terminal app
  3. Paste the following into terminal:

    rm -rfv /Users/$(id -un)/Library/Caches/com.logmein.ignition/fsCachedData/*



    It should look like this (terminal may be white instead):

  4. Now, re-open the LogMeIn Client
  5. In the top-left of your screen, click "LogMeIn Client", then Preferences:
  6. Clear the saved credentials:
  7. Now, login again and it should work!
Note! If you are confused at any point, or if the steps do not work, please reach out to a SME.

"GoToMyPC Cannot Be Started At This Time"

Explanation:

This occurs when Windows reports that GoToMyPC viewer files are "in-use"

Solutions:

  • This works the majority of the time
    • Note! Continue to the next steps only if the user reports that this happens to them frequently
  • Download the Permissions Fix Tool
  • Right click the download and extract it
  • Right click the GoToOpener_PermissionsFix.bat file, and select Run as administrator
  • A window should quickly appear and disappear
  • Attempt to connect to the computer again

Running the Script

  • Download the GoToMyPC_Client_Fixit.bat tool from
  • Double-click it to run
  • Press Enter to begin & allow it to run
  • Note! After it is done running, it will ask whether you would like to reboot the machine
    • If the customer is okay with rebooting, you may press enter to reboot the machine
    • Otherwise, click the 'x' on the window and allow them to save their work / reboot later

Creating desktop shortcuts

  1. Connect to the host machine via the website
  2. Once in session, click File → Create shortcut to...
  3. Click Yes to continue
  4. Have them enter their GoToMyPC account password
  5. Click Continue then OK
Explanation: The desktop shortcut does not use the GoToOpener in order to connect to the session. If a customer uses this method to connect to the machine, it is faster and will avoid this problem in the future.
