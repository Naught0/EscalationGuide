Hamachi assigns 0.0.0.0 IP addresses to all peers

Explanation:

This can sometimes occur when Windows does not properly "bridge" the Hamachi adapter with the Internet adapter. This can happen at installation, during Windows updates, or randomly.

Solution: Rebridge the adapters

  1. Open the run prompt Win + r
  2. Enter ncpa.cpl to open Network Connections
  3. You should see an adapter named Hamachi as well as Hamachi - ###-###-###
    4. Pro tip: You can tell the adapter is not bridged correctly if their Internet adapter does not say Enabled, Bridged
  4. The Hamachi - ###-###-### adapter is bridged with their regular internet adapter (typicaly labeled Ethernet or WiFi)
  5. To re-bridge these adapters, hold Ctrl and click both Hamachi - ###-###-### and Ethernet (or similar)
  6. Now, right click one of the adapters and select "Remove from bridge"
  7. Repeat step 5, this time right-clicking and selecting Bridge Connections
  8. The internet will likely disconnect temporarily at this point, wait for it to come back
  9. Re-test
Note! If you receive an error at any point, or this does not resolve the 0.0.0.0 IP issue, reach out to a SME.
