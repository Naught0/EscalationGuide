LogMeIn Pro
  • First, enable debug-level logging
    1. Open the LogMeIn Control Panel on the host machine
    2. Navigate to Options → Preferences → Advanced
    3. Check off Enable debug-level logging and press OK
  • Important! Recreate the issue you are escalating after you have enabled debug
  • Now, to collect the log files
    1. Open Windows Explorer
    2. In the address bar, type %programdata%, and press Enter
    3. Right click the LogMeIn folder → Send to → Compressed (zipped) folder
    4. Use the file-transfer option in LogMeIn Rescue to transfer the LogMeIn.zip file to your machine
  • Now, upload the file to your SalesForce case
  • Be sure to fill out the escalation format in your case notes
  • Once you have finished your case do not close it! —change the case owner to IAM Sage Advanced

LogMeIn Pro Client
  • First, open the LogMeIn Client
  • Select View → Options → Enable debug-level logging
  • Important! Be sure to recreate the issue before moving to the next steps
  • Now, collect the logs:
    1. Open up Windows Explorer (Win + E)
    2. At the top, type: %localappdata%
    3. In the list of folders, find LogMeInIgnition
    4. Right-click the folder and choose Send to → Compressed (zipped) folder
    5. Move the file to the Desktop folder for easy access
  • Now, use Rescue's File Manager to transfer the file to your machine
  • Upload the files to your SalesForce case
  • Be sure to fill out the escalation format in your case notes
  • Once you have finished your case do not close it! —change the case owner to IAM Sage Advanced

LogMeIn AV
  • First, enable debug-level logging
    1. Open the LogMeIn Control Panel on the host machine
    2. Navigate to Options → Preferences → Advanced
    3. Check off Enable debug-level logging and press OK
  • Important! Recreate the issue you are escalating after you have enabled debug
  • Now, gather the Support Tool logs
    1. Download the LogMeIn AV Support Tool on the customer's machine. You can find it
    2. Once the tool is downloaded, double-click it and allow it to run

      3. Note! The tool will run silently (you won't see it doing anything)

      It will place a zipped folder on the customer's desktop

  • Once the tool has finished running, collect the debug-level logs and Support Tool logs
  • To collect the debug logs
    1. Open Windows Explorer
    2. In the address bar, type %programdata%, and press Enter
    3. Right click the LogMeIn folder → Send to → Compressed (zipped) folder
    4. Use the file-transfer option in LogMeIn Rescue to transfer the LogMeIn.zip file to your machine
  • To collect the support tool logs
    • Use the file-transfer option in LogMeIn Rescue to transfer the supporttoolXXXXX.zip from their desktop
  • Now, upload the files to your SalesForce case
  • Be sure to fill out the escalation format in your case notes
  • Once you have finished your case do not close it! —change the case owner to IAM Sage Advanced

LogMeIn Hamachi
  • Hamachi logs are a bit tricky to get to
  • They are located in C:\Windows\ServiceProfiles\LocalService\AppData\Local\LogMeIn Hamachi
  • Important! You will need to manually navigate to this location
    1. Open Windows Explorer
    2. Select View → Hidden items (click the checkbox)
    3. Start in C:\Windows and click each folder in the path

      4. ServiceProfiles → LocalService → AppData → Local → LogMeIn Hamachi

      Note!

      You will likely receive admin prompts clicking on each of these folders—just click OK

  • Now, locate the file that says h2-engine or h2-engine.log

  • Drag this file to the desktop
  • Use the file-transfer feature in LogMeIn Rescue to transfer the file to your machine
  • Upload the file to your SalesForce case
  • Be sure to fill out the escalation format in your case notes
  • Once you have finished your case do not close it! —change the case owner to IAM Sage Advanced
  • Hamachi logs can be gathered directly through your Rescue Session pretty easily
    1. Open the File Manager in LogMeIn Rescue
    2. Navigate to / → Library → Logs → LogMeIn Hamachi
    3. Gather the files called h2-engine.log, stderr.log, and stdout.log
  • Once the files are on your machine, upload them to your SalesForce case
  • Be sure to fill out the escalation format in your case notes
  • Once you have finished your case do not close it! —change the case owner to IAM Sage Advanced

GoToMyPC
  • Important! In order to gather logs, there must be no active sessions
  • Ensure you are directly connected to the computer that is experiencing the problem before continuing
  • To collect GoToMyPC logs
    1. Download GoToLogFiles on the customer's computer

    2. Right click the downloaded file and select Extract All...

    3. Now double-click the GoToLogFiles file to run it

    4. Check off any item that says "GoToMyPC"

    5. Click Archive logs to ZIP
  • The logs will automatically be collected and placed on the customer's desktop

  • Collect the GoToLogFilesXXXXX.zip file from the customer's desktop using Rescue's file-transfer option

  • Upload the file to your SalesForce case
  • Be sure to fill out the escalation format in your case notes
  • Once you have finished your case do not close it! —change the case owner to IAM Sage Advanced
  • Important! Make sure there are no active GoToMyPC sessions when you collect logs
  • Ensure you are directly connected to the computer that is experiencing the issue
  • To collect the Mac logs
    1. Open the Terminal app
    2. Copy and paste the below, follwed by Enter:
      3. sw_vers > /tmp/sysinfo.txt; zip -r ~/Desktop/GoToLogFiles$(date +%s).zip "/Library/Logs/com.logmein.GoToMyPC" "/Library/Logs/com.logmein.GoToOpener" "/tmp/sysinfo.txt"
